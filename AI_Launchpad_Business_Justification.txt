Q1. Does your app need a database or storage bucket?
--------------------------------------------------------------------------------

Yes.

The application uses a local SQLite database as an intermediate cache between
BigQuery and the UI. For Launchpad deployment, this needs to be replaced with
either Cloud SQL (PostgreSQL) or a Cloud Storage bucket to persist data across
container restarts.

Database details:
  - Schema   : 3 tables (roster, roster_overrides, refresh_log)
  - Data size : Approximately 80,000 rows, less than 50 MB
  - Access    : Read/write from the application only

The app does NOT require any blob or media storage. All exports (CSV, Excel,
PDF) are generated in memory and streamed directly to the user's browser.


--------------------------------------------------------------------------------
Q2. Does your app require Element LLM Gateway access?
--------------------------------------------------------------------------------

No.

This application is a pure data visualization and roster management tool. It
does not use any Large Language Models, generative AI, or AI inference of any
kind. All logic is deterministic. Data is fetched from BigQuery, transformed
into a schedule matrix, and rendered as HTML tables with export options.

No LLM Gateway access is needed now or planned for future phases.


--------------------------------------------------------------------------------
Q3. External or internal API integrations needed?
--------------------------------------------------------------------------------

Yes. One internal GCP integration is required. No external APIs are used.

  Integration  : Google BigQuery
  Type         : Internal - Walmart GCP
  GCP Project  : wmt-cc-datasphere-prod
  Datasets     : CS_AGNT_SCHED, CS_AGNT
  Access type  : Read-only SELECT query
  Frequency    : Runs automatically every 2 hours via APScheduler
  SDK used     : google-cloud-bigquery (Python)
  IAM required : roles/bigquery.dataViewer on wmt-cc-datasphere-prod

All traffic stays within the Walmart GCP environment. No external internet
APIs are called at any point during application runtime.

Q5. What is the expected benefit of this Application?
--------------------------------------------------------------------------------

PROBLEM BEING SOLVED

The CES India transportation team currently manages agent shift schedules
through manually updated Excel files shared over email and SharePoint. This
creates a fragmented, error-prone process where outdated copies circulate,
schedule changes are missed, and supervisors have no single source of truth
for upcoming or historical rosters.

DIRECT BENEFITS

1. Single Source of Truth
   All shift data is sourced directly from BigQuery on the wmt-cc-datasphere-prod
   project. No more stale Excel files being shared manually across the team.

2. Real-Time Visibility
   The application auto-refreshes every 2 hours and provides a 30-day future
   schedule view and a 90-day historical view, accessible to the full team
   instantly from any browser on the Walmart network.

3. Operational Efficiency
   Supervisors and L1/L2 managers can view their team roster, export it in
   CSV, Excel, or PDF format, and share it in seconds instead of manually
   compiling reports each time.

4. Admin Override Capability
   Admin users can manually adjust shift assignments directly in the UI.
   Overrides are highlighted visually with audit notes and timestamps,
   and are visible immediately to all viewers.

5. Ad-Hoc Roster Generation
   Admin can select any group of associates, set a custom shift and date
   range, choose week-off days, and instantly download a formatted roster
   in Excel or PDF. This previously took hours of manual effort in Excel.

6. Reduced Escalations
   With schedule visibility available self-serve for 200+ associates,
   transport coordination queries to the analytics team are expected to
   reduce significantly after go-live.

7. Auditability
   Every override is logged with timestamp and modifier name. Every GCP
   refresh is logged with record count and status. Full history is
   available at all times within the application.

ESTIMATED IMPACT

  Associates covered  : ~200 CES IND associates across multiple LOBs
                        (Fraud, Chargebacks, Corp Gift Card, Pick Up/OLG,
                        Account Review, and others)

  Supervisors impacted: ~8 to 10 L1 and L2 supervisors who currently spend
                        30 to 60 minutes per week manually compiling and
                        sharing rosters. This application eliminates that
                        effort entirely.

  BigQuery cost impact: Zero incremental cost. The query runs read-only on
                        an existing dataset already licensed for reporting
                        under wmt-cc-datasphere-prod.
